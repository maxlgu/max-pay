Max Pay experiments

Max Pay

Experimental Apps

  • max-nonbasiccard: an all-inclusive non-basic card payment handler.
  • external-origin-http-content: a payment app that contains http content of an external origin.
  • external-origin-https-content: a payment app that contains https content of an external origin
  • iframe: a payment app that contains an iframe tester.
  • iframe-external-http-origin: a payment app that contains an iframe of an external http origin
  • iframe-external-https-origin: a payment app that contains an iframe of an external https origin
  • iframe-local-https-origin: a payment app that contains an iframe of a local origin
  • insecure-content-after-onload: a payment app that insert an image of an http origin after onload.
  • load-end-security-check: a payment app that has contains an image of an external http origin but does not trigger insecure content alert because the app uses long polling to block the page from finish. This app demos how circumventing the security check is possible.
  • navigation: a payment app that contains a navigation tester.
  • open-in-new-tab: a payment app that opens a url in a new window.
  • open-window-external-origin: a payment app that calls openWindow for an external https url as a payment app url.
  • open-window-nonexistent-url: a payment app that calls openWindow for an non-existent https url as a payment app url.
  • permissions: a payment app that request permissions
  • redirect: a payment app that redirects itself to a http url.
  • redirect-same-origin: a payment app that redirects itself to a url of the local origin.
  • theme-color: an app that paints the toolbar into black.
  • gpay-fail: merchant fails the gpay.
  • gpay: the GPay2.0 payment app.
  • pmc: the app to test PaymentMethodChange event.
  • cookie: the app to test cookie.
  • intersection-observer: intersection observer api tester
  • maxpay-bobflip: the Bobpay app that flips the position of buttons.
